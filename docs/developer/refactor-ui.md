# 交互与视觉体验设计师的防遮挡体验升级方案

为了彻底解决 `QuickButtons`（快捷按钮列）和 `MainPanel`（主面板）作为强悬浮组件对宿主页面的遮挡痛点，我们可以从**第一性原理**出发，剖析“遮挡”的本质：**两个维度的视觉元素在同一个 Z 轴高度和 XY 平面发生了资源争夺**。

在复杂多变的 AI 对话站点中，我们很难通过强行修改原页面（Push/Padding 挤压）来让出空间（容易导致各站点的适配灾难）。因此，我们的解决核心必须聚焦于：**时间换空间（动态显隐）**、**形态降级（从面到点）** 以及 **焦点感知（智能避让）**。

结合现有的代码基础（目前已具备拖拽、基础吸附、透明度调节等），以下是我作为交互与视觉体验设计师，为您提供的一套**系统化防遮挡体验升级方案**：

### 1. 幽灵模式 + 事件穿透 (Ghost & Pass-through Mode) 👻

这是对于干扰最低的方案，核心在于“当不需要时，它应该仿佛不存在”。

- **视觉幽灵化**：在当前代码 `quickButtonsOpacity` 的基础上，增加“极致空闲状态”。当鼠标离开组件且没有交互超过 5 秒时，组件透明度降至极低（如 `0.15` 或纯线条轮廓态）。
- **动作感知（滚动隐藏）**：监听全局鼠标滚动（`wheel` 或宿主 `scroll`）。当用户正在聚精会神阅读/滚屏时，`QuickButtons` 立刻平滑平移出屏幕边缘或完全隐藏；滚动停止 0.8 秒后，再优雅地浮现。这能极大提升沉浸感。
- **按键一键穿透**：引入一个“透视快捷键”（例如长按 `Alt` 或 `Ctrl`）。按住时，全局 Ophel 面板变为半透明且 `pointer-events: none;`。这样即便面板挡住了底层 AI 网站的某个核心按钮，用户只需按住快捷键就可以**直接穿透点击到底层网页**，松开立刻恢复。这种微交互会让高级用户拍案叫绝。

### 2. 形态降级与液态折叠 (Liquid Collapse) 💧

把工具栏从“一根棍子”降级为“一颗水滴”，把面板从“一面墙”降级为“一根胶囊”。

- **QuickButtons 智能引力胶囊**：如果用户觉得竖向一排按钮还是占地方，可以让这一排按钮在空闲时像水一样“聚合”缩短，融合成一个孤立的、发光的 `SparkleIcon`。当视线/鼠标距离该图标 50px（引力场）以内时，它再像多级火箭一样弹性展开成完整的按钮条。
- **边缘潜水 (Edge Peeking 的进阶)**：目前 `MainPanel` 代码中已预留了 `isEdgePeeking` 和 `edgeSnap` 的概念。可以进一步强化：当用户把面板拖到屏幕极限边缘时，它不只是“贴边”，而是 80% 的躯体“潜入”屏幕外，只留下一条带有毛玻璃发光反馈的“脊背”（竖向 Indicator）。点击这条脊背，面板像抽屉一样平滑弹出；失去焦点即缩回。

### 3. 上下文感知与活体避让 (Context-Aware Collision Avoidance) 🧠

最高级的 UI 感觉不到被设计，它像一个懂事的助理。

- **DOM 碰撞侦测（活体避让）**：主流 AI 站点（如 ChatGPT, Claude）的内容区通常是居中定宽的，两侧大片留白。我们的组件可以利用 `IntersectionObserver` 或者是读取屏幕中间宽度的策略，计算出一个“安全区 (Safe Zone)”。如果有弹出弹窗（比如系统的 Toast 或宿主站点的 Dialog），或者是响应式视口发生变化，`QuickButtons` 能够检测到“我在遮挡有文字的地方”，主动如活物一般产生位移，避开高文字密度区域。
- **动态调整吸附偏好**：例如检测到左侧边栏展开了，右侧的按钮如果觉得拥挤，应该智能提示或者自己微调悬浮高度，避免遮挡原生发送消息框的核心区域。

### 4. 极致毛玻璃与层级融合 (Hyper Glassmorphism) 🪟

从代码 `var(--gh-glass-bg)` 可以看出，您已经引入了玻璃拟态。但这还不够。

- **磨砂景深 (Depth of Field)**：不仅要做 `backdrop-filter: blur()`, 而且应当结合亮度动态调节。如果识别到下方被遮挡的是黑色块，面板内部可以智能提亮；如果是白色，则加深阴影边界和深冷色玻璃反光。
- **无边框与柔性阴影**：让 `MainPanel` 和 `QuickButtons` 在非 hover 状态下去除生硬的 `border`，转而使用一层极其微弱的光晕（`box-shadow: 0 0 15px currentColor` 的低透明度版），让它看起来像是漂浮在网页上方的一层“雾”，而不是一块“硬纸板”。这种视觉错觉能有效降低“视线被遮断”的焦虑感。

### 5. 迷你外挂形态：Mini-bar 吸附模式 🚀

对于 `MainPanel` 这样一个巨型空间消耗者。

- **形态变异切换**：在 header 上增加一个“Mini”模式切换按钮。当用户只想要看看大纲或者提示词，不想要占据巨大屏幕空间时，`MainPanel` 从“卡片形态”骤缩为一条横向的、可以任意拖拽吸附在屏幕顶部或底部的 Mac 风格的 Touch Bar (Mini-bar)。在迷你条中，使用横向微缩卡片或者跑马灯展现当前重要信息，完全将阅读视野归还给用户。
