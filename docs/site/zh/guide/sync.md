# 备份与同步

Ophel 的所有数据默认只保存在你本机的浏览器存储中。本页介绍如何将数据导出备份，以及如何通过 WebDAV 在多台设备之间同步。

## 本地备份（导入/导出）

**设置 → 备份与同步**

支持将数据导出为 JSON 文件，以便备份或迁移到另一台设备：

| 导出类型 | 包含内容                                                                        |
| -------- | ------------------------------------------------------------------------------- |
| 完整备份 | 所有数据：设置、提示词、会话元数据、文件夹、标签（及可选的 Claude Session Key） |
| 仅提示词 | 提示词库                                                                        |
| 仅设置   | 所有配置项                                                                      |

**导入**：将 JSON 文件拖入或点击选择文件导入。Ophel 会在写入前对格式做校验，导入后自动刷新页面以使配置生效。

> 导入会**覆盖**现有数据，建议在导入前先导出一份当前备份。

## 从 Gemini Voyager 导入

**设置 → 备份与同步 → 从 Gemini Voyager 导入**

[Gemini Voyager](https://chromewebstore.google.com/detail/gemini-voyager/pgmfbpioicfjbfghalhcjkfdlkadamep) 是另一款 Gemini 会话管理扩展。如果你早先用它整理了文件夹结构，可以直接将其导出结果导入到 Ophel。

**操作步骤**：

1. 在 Gemini Voyager 中导出文件夹数据（JSON 格式，文件名形如 `gemini-voyager.folders.v1...`）
2. 复制该 JSON 文件的内容
3. 在 Ophel 设置中找到「从 Gemini Voyager 导入」，将内容粘贴到输入框并点击「开始导入」
4. Ophel 会呈现预览（新增文件夹数、新增会话数等），确认后执行导入

**导入行为说明**：

- **增量合并**：导入不会覆盖你在 Ophel 已经整理好的文件夹。只有在收件笱或之前导入过的 Voyager 文件夹中的会话才会被重新归类
- **文件夹压平**：层级嵌套的 Voyager 文件夹会被压平成路径名（例如「父级 / 子级」），不支持层级结构
- **只适用于 Gemini**：导入功能使用当前浏览器中登录的 Gemini 账号，请在 Gemini 页面打开设置执行导入

## 清空所有数据

**设置 → 备份与同步 → 清空所有数据**

删除 Ophel 存储的全部本地数据——包括提示词、会话元数据、文件夹、标签和所有设置，恢复到安装初始状态。

::: warning 不可撤销
此操作无法撤销。如需保留数据，请先导出备份。
:::

## WebDAV 同步

WebDAV 同步让你把 Ophel 数据上传到自己的私有服务器，并在多台设备之间保持一致。常见的 WebDAV 服务器包括 Nextcloud、坚果云（WebDAV 接口）、Synology NAS 等。

**设置 → 备份与同步 → WebDAV**：

| 配置项       | 说明                                                      |
| ------------ | --------------------------------------------------------- |
| 服务器地址   | WebDAV 服务器的完整 URL（以 `https://` 开头）             |
| 用户名       | WebDAV 账号用户名                                         |
| 密码         | WebDAV 账号密码（存储在本机，不上传给任何第三方）         |
| 远端目录     | 同步文件存放的路径（默认 `ophel`，自动创建）              |
| 同步模式     | 手动（点按钮）或自动（按设定间隔）                        |
| 自动同步间隔 | 自动模式下，两次同步之间的最短间隔（分钟）                |
| 数据来源     | 选择要同步的数据类型（设置 / 会话 / 提示词 / Claude Key） |

### 什么数据会被同步？

WebDAV 同步的是 Ophel 在本地维护的**元数据**：

- 你在 Ophel 中设置的文件夹、标签、置顶状态
- 提示词库
- Ophel 设置项
- Claude Session Key（可选，默认包含，共享备份文件时请注意）

**不会同步**的内容：AI 平台上的实际对话消息——那些内容属于各 AI 服务，Ophel 无权访问和上传。

### 同步注意事项

- WebDAV 同步依赖可选权限 `<all_urls>`。首次启用会弹出浏览器权限请求。
- 多设备场景建议使用"手动同步"，避免多端同时写入产生冲突。
- 若要在新设备上恢复数据，先配置 WebDAV 服务器信息，再点击"从服务器拉取"。

## 权限管理

**设置 → 权限**

Ophel 使用以下权限：

| 权限            | 类型 | 用途                          |
| --------------- | ---- | ----------------------------- |
| `storage`       | 必需 | 保存本地配置与数据            |
| `notifications` | 可选 | 生成完成桌面通知              |
| `cookies`       | 可选 | Claude Session Key 读取与切换 |
| `<all_urls>`    | 可选 | WebDAV 同步 + 去水印功能      |

每项可选权限都可以在此页面单独撤销。撤销后，依赖该权限的功能会自动停用。
