# Claude 账号管理

Claude 每个账号都有使用频率限制，达到上限后只能等待重置。如果你有多个 Claude 账号，Ophel 的账号管理功能让你把多个账号的 Session Key 存储在扩展中，用完一个账号时一键切换到下一个，延续对话不间断。

## 什么是 Session Key？

Session Key（`sk-ant-sid...` 格式）是你的浏览器会话在 claude.ai 的认证令牌。Ophel 通过拦截请求并注入指定的 Session Key，让你在不登出登入的情况下切换账号。

Session Key 存储在本机的浏览器扩展存储中（`chrome.storage.local`），Ophel 不会将其上传到任何服务器。

## 添加 Session Key

**设置 → 站点设置 → Claude**

有两种方式添加：

**自动导入（推荐）**：点击**导入当前会话**，Ophel 自动从浏览器 Cookie 中读取当前登录账号的 Session Key，直接填入，无需手动复制。

**手动添加**：点击添加按钮，粘贴 Session Key 字符串（格式为 `sk-ant-sid...`）。

每个 Key 可以添加一个备注名称，方便区分是哪个账号。

## 切换账号

切换方式有两种：

- **工具箱菜单** → 直接点击当前激活的 Key，选择另一个
- **快捷键 `Ctrl+Alt+S`**（Mac: `Cmd+Option+S`）— 切换到 Key 列表中的下一个账号

切换后 Ophel 会刷新当前 Claude 页面，使新 Key 生效。

## 管理 Key 列表

| 操作       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 设为当前   | 将某个 Key 设为当前使用的账号                                |
| 单个测试   | 发送一次轻量请求验证该 Key 是否有效（未达到限额且凭据正确）  |
| 全部测试   | 批量验证所有 Key，按 500 ms 间隔逐一测试，避免触发频率限制   |
| 导出       | 将全部 Key 导出为 JSON 文件（备份用）                        |
| 导入       | 从 JSON 文件恢复 Key 列表                                    |
| 删除       | 删除单条记录，不影响 Claude 平台上的账号本身                 |

::: warning 注意共享风险
导出的 JSON 文件包含完整的 Session Key，任何拿到文件的人都可以使用你的账号配额。分享 WebDAV 备份或本地文件时请特别注意。
:::

## 安全性说明

- Key 仅存储在本机，不发往 Ophel 的任何服务器
- Ophel 的网络请求均通过 claude.ai 的官方 API 发出，不经过任何代理
- 如果某个账号被封或 Key 失效，Ophel 也无法帮你绕过；Key 测试功能会标记出无效的 Key

## 相关快捷键

> Mac 用户：**Ctrl** = **Cmd**，**Alt** = **Option**。

| 操作             | 快捷键       |
| ---------------- | ------------ |
| 打开 Claude 设置 | `Ctrl+Alt+C` |
| 切换到下一个 Key | `Ctrl+Alt+S` |
