# 用量统计

对于有每日使用次数限制的 AI 服务，Ophel 提供一个本地计数器，帮助你追踪今天已经用了多少次以及大概消耗了多少 Token。

## 功能说明

用量统计完全在本地运行，不调用任何外部 API，也不依赖平台提供的用量数据——Ophel 通过观察页面上的消息来自行计数和估算。

计数器会显示在支持站点的输入框附近，包含：

- 今日对话轮次（发送次数）
- 估算输入 Token 量
- 估算输出 Token 量
- 已用次数 / 每日限额的进度条（可选）

> Token 估算基于简单的字符统计，与平台实际计费方式可能有出入，仅供参考。

## 启用方式

**设置 → 功能 → 用量统计**：

| 设置项   | 说明                                     | 默认 |
| -------- | ---------------------------------------- | ---- |
| 启用     | 是否显示用量统计小组件                   | 关闭 |
| 每日限额 | 你的预期每日使用上限（用于进度条可视化） | 100  |
| 自动重置 | 每天 0 点自动将计数清零                  | 关闭 |

## 适用场景

- Gemini Advanced 等有配额限制的订阅服务
- 需要在多个账号之间分配用量的场景
- 想大致掌握自己 AI 使用习惯的用户

用量统计是纯参考工具，无法阻止你超出平台限额，也不代表平台实际的计费或配额消耗。
